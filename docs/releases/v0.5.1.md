# ptv 0.5.1

Patch release to tolerate upstream data quality issues in the Transport Victoria
GTFS feed that prevented `ptv gtfs update` from completing.

## Bug fixes

- **GTFS ingestion tolerates duplicate `trip_id` rows.** The Regional Coach feed
  currently contains a duplicate `trip_id` in `trips.txt`. The compiler now skips
  the duplicate (keeping the first occurrence) and prints a warning instead of
  aborting the entire feed import.
- **GTFS ingestion tolerates non-increasing stop-time segments.** The Regional
  Coach feed contains a small number of stop-time segments where arrival at the
  next stop precedes departure from the current stop or `stop_sequence` is
  non-monotonic. These segments are now excluded from the connection graph with a
  warning instead of failing ingestion.

## Known issues

Both workarounds above address symptoms of upstream feed quality, not root
causes. TODO comments in `internal/gtfs/compiler_csv.go` flag these for
investigation and potential upstream reporting.

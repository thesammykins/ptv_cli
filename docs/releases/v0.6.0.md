# ptv 0.6.0

This release adds credential-free offline timetable planning and bounded static
PTV v3 enrichment while preserving the existing live API contracts.

## Highlights

- Adds validated GTFS Open Data generations with generation-local integer keys,
  namespaced public identifiers, Melbourne service-day handling, and local
  timetable/trip queries.
- Adds the `timetable` and `trip` commands plus GTFS-first search, stops, lines,
  station, next, disruptions, and route views.
- Normalizes Transport Victoria GTFS-Realtime trip updates, alerts, and vehicle
  positions with deterministic freshness, cache, rate-limit, and degraded
  output behavior.
- Adds a reviewed v3 snapshot containing only slowly changing outlet and
  station-facility data. Route and timetable data is never served from the
  snapshot, avoiding stale routing results.
- Keeps additive JSON source and warning fields, stable empty collections,
  identifier namespaces, and explicit upstream attribution/disclaimer data.
- Adds a pinned monthly snapshot-update workflow that can be enabled with
  repository secrets without putting credentials in the distributed binary.

## Migration and compatibility

- Run `ptv gtfs update` once after installing 0.6.0 to publish a schema-v2
  generation for offline planning.
- Timetable API and Transport Victoria Open Data credentials remain separate.
  Core GTFS planning and static inspection do not require Timetable API
  credentials.
- Existing live commands retain their prior top-level JSON fields; new
  `data_source`, `source_notice`, `freshness`, and `warnings` fields are
  additive.

## Data licensing

The bundled v3-derived data is distributed under the PTV Timetable API Data
Creative Commons Attribution 4.0 International Licence. Attribution,
modification, disclaimer, and non-endorsement notices are included in the
repository `NOTICE` file and release archives.

## Known limitations

- GTFS and GTFS-Realtime coverage, identity quality, and freshness remain
  dependent on Transport Victoria's upstream feeds.
- The bundled snapshot intentionally excludes route, direction, and timetable
  data; refresh it when the reviewed static enrichment source changes.

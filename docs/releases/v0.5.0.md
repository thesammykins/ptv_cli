# ptv 0.5.0

This release makes local journey planning substantially faster and strengthens
the GTFS, PTV Timetable, and GTFS-Realtime contracts used by the CLI.

## Highlights

- Replaces whole-feed planning work with validated immutable GTFS generations,
  materialized connections, bounded directional windows, and multi-label CSA.
- Enforces GTFS transfer types 0–5, pathway direction, pickup/drop-off policy,
  linked-trip continuations, service instances, and Melbourne service-day/DST
  rules.
- Normalizes PTV and GTFS-Realtime output while preserving identifier
  namespaces, freshness evidence, JSON stdout, warning stderr, cancellation,
  and bounded upstream errors.
- Corrects fare JSON to use the endpoint's real `FareEstimateResultStatus`
  contract instead of emitting a fabricated empty generic `status` object.
- Diagnoses route/stop mismatches for filtered departure queries instead of
  returning an unexplained empty success; legitimate no-departure results stay
  valid.
- Marks upstream zero route-stop sequences as unsequenced in human tables while
  preserving JSON compatibility.
- Uses the documented single `KeyID` contract for Transport Victoria Open Data
  GTFS-Realtime requests.
- Allows local GTFS planning and static-data inspection without unrelated PTV
  Timetable credentials.
- Adds `--allow-conditional` for callers that explicitly choose GTFS
  pickup/drop-off policy values 2 and 3; conservative exclusion remains the
  default.
- Adds pinned Go 1.25.12 mise workflows for build, vet, tests, race detection,
  host-timezone checks, CGO-disabled builds, and Windows cross-compilation.

## Performance

On the disclosed comparable production snapshot, the audited Flinders Street
to Box Hill cases improved from 68.57 seconds to a 0.97-second median for
forward planning and from 48.87 seconds to a 1.08-second median for arrive-by.
Maximum observed RSS was 154.75 MiB and 162.89 MiB respectively.
The deliberately uncommon Southern Cross–Albury case exercised the expanded
six-hour window at 4.86 seconds and 425.59 MiB; this remains a documented
tradeoff rather than the common interactive path.

## Migration and compatibility

- Existing legacy GTFS databases cannot provide the new routing invariants.
  Run `ptv gtfs update` once after installing 0.5.0 to publish a schema-v2
  generation; an old usable database is not modified in place.
- `--json` is normalized output rather than a raw upstream passthrough. Shared
  time, warning, URL, and identifier rules are documented in
  [`docs/json-contract.md`](../json-contract.md).
- Timetable API and Open Data credentials are separate capabilities. Existing
  environment and keyring names remain supported.

## Validation

The candidate passed the repository's full `mise run audit-check`, production
GTFS planning benchmarks, owner-authorized live PTV and GTFS-Realtime probes,
an independent adversarial review, and the documented
[version-stamped candidate dogfood](v0.5.0-dogfood.md).
